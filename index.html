What Are Image Hover Effects?

Image hover effects are specific css effects that are added to elements via the ':hover' selector. In simpler terms, it allows for dynamic changes in a website when a certain image is hovered over. For a very basic examples, you can look at the images in "Gallery", which grow in size slightly when hovered over(when on desktop view). Just as the ':hover' selector can be used to modify text when its hovered over, images work in the same way.

When applying the hover image effects to my Gallery, I tried to apply the knowledge I had of text hover effects(from the links in the Navbar and Footer), with a high level of success. I marked the specific image elements with a particular class within my html, then applied css changes to that class, given the ':hover' selector. I simply changed the scale of the image slightly(105%) when the images are hovered over to give a small amount of user feedback.

Image hover effects are a relatively easy way to add interactivity and responsiveness to a website without(necessarily) vastly changing its function. They can vary from very minor cosmetic effects, to complex animations, to function-relevant changes. Although there is always a risk of overdoing it, adding image hover events to a website effectively can vastly improve it, both in terms of cosmetics and functionality.

Because of the simplicity that the initial implementation held, I decided to look into more interesting effects to apply to the images. I looked into image filters(which include brightness, sepia, contrast, hue, and blurs), as well as image transformations(which include translations, rotations, skews, and scaling).

Below this Blog is three yellow squares that can be hovered over to produce effects. Tapping/Clicking the boxes will also produce the same effect, for those who are on devices that cannot hover over an image. The first one becomes partially transparent when hovered over, utilizing the opacity filter. In reality, because of the simplicity of the example, the square just becomes a light purple color(as yellow being partially opaque on top of purple makes a lighter purple). The second square is skewed along both the X-axis, and the Y-axis when hovered over, which vastly changes its shape. Finally, the third box is simply rotated 45 degrees when hovered over.

The only issues I ran into when attempting to implement these effects were unexpected behaviors with the 'matrix' transformation. This transformation is a combination of scale, skew, and translate, on both the X-axis and Y-axis, meaning that it takes in 6 values. Because the order of these parameters aren't entirely intuitive(scaleX, skewY, skewX, scaleY, translateX, translateY), when experimenting, I often found myself misplacing values, leading to a number of strange effects. Once, by mistakingly setting scaleY to 0, I discovered that the square would rapidly flash when hovering over it. Hovering over the box would cause it to disappear(because its scaleY = 0), which would mean that it is no longer being hovered over, causing it to go back to normal, before starting the loop again. A similar, although less extreme version of this can be observed when hovering over the second square's top right or bottom left corners, as the skew effect changes the bounds of the box.

Even these image hover effects are only scratching the surface, as animations and effect timings can be created through css, allowing for an extreme level of creative expression, beyond simple filters, transformations, and style changes.
